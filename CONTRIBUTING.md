# Contributing

Issues and pull requests are welcome.

1. Fork the repository and create a focused branch.
2. Run `pnpm install`.
3. Make the change and add or update tests.
4. Run `pnpm check:all` before opening a pull request. This includes TypeScript, unit, production-build, and Chromium end-to-end checks.

Please keep generated builds, recordings, local environment files, and credentials out of commits.

A note on naming: the simulation core is ported from [Fluoddity](https://github.com/aphid91/Fluoddity) (see [THIRD_PARTY_NOTICES.md](THIRD_PARTY_NOTICES.md)), so a few identifiers deliberately keep that name as provenance markers — the `fluoddity-core-*` rule/sensor tags in `realtimeGpuSim3d.ts` and the upstream file citations in `livePresets.ts`. Everything project-specific uses the `lifesim` name, including the `window.__lifesim*` automation hooks the test suites drive.

Preset JSON files are generated by the in-app `Save JSON` button rather than written by hand — to contribute a preset, craft the look in the app and export it.
